Three pillars.
One prediction engine.

The pipeline is simple: clean corpus, build n-gram tables, score with frequencies, and serve top-k suggestions in real time. Every result is traceable to corpus counts.

Corpus Indexing

Documents are cleaned, tokenized, and indexed into unigram, bigram, and trigram tables. The same corpus always produces the same counts.

Probability Scoring

Each candidate is scored with conditional probability: P(next | context) = count(context, next) / count(context). This drives the ranking.

context "machine"
count(context) 500
count(context, next) 142
P(next | context) 0.284

Top-k Suggestions

Top candidates are returned with probabilities. Clicking a suggestion appends it to the query for fast iteration.

§ 02 Lifecycle

From query to prediction.
Corpus-anchored results.

Every request moves through four steps. Text is normalized, n-grams are retrieved, and probabilities are scored. Each suggestion is traceable to corpus counts.

01
Capture
User enters one or two words. The request is timestamped and queued for lookup.
STATE · CAPTURED
02
Normalize
Input is lowercased, tokenized, and trimmed to the latest context window.
STATE · NORMALIZED
03
Retrieve
N-gram index is searched. Candidate next words and frequencies are gathered.
STATE · RETRIEVED
04
Deliver
Probabilities are computed and top suggestions are returned to the interface.
STATE · DELIVERED
