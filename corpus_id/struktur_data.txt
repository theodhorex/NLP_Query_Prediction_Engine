Struktur data adalah cara pengorganisasian dan penyimpanan data dalam komputer sehingga data dapat diakses dan dimanipulasi secara efisien. Pemilihan struktur data yang tepat sangat penting dalam pengembangan perangkat lunak karena dapat mempengaruhi performa, penggunaan memori, dan kemudahan implementasi algoritma. Struktur data diklasifikasikan menjadi dua kategori utama yaitu struktur data linear dan struktur data non-linear. Struktur data linear menyusun data secara berurutan seperti array, linked list, stack, dan queue. Array adalah kumpulan elemen dengan tipe data yang sama yang disimpan di lokasi memori berurutan dan diakses menggunakan indeks numerik. Array memiliki kompleksitas akses O1 tetapi ukurannya tetap dan tidak dapat diubah setelah dideklarasikan. Linked list adalah kumpulan node yang masing-masing berisi data dan pointer ke node berikutnya. Linked list memungkinkan penyisipan dan penghapusan elemen secara dinamis dengan kompleksitas O1 di posisi yang diketahui, namun akses acak membutuhkan waktu On. Stack adalah struktur data yang mengikuti prinsip last in first out atau LIFO dimana elemen yang terakhir dimasukkan adalah yang pertama dikeluarkan. Stack digunakan dalam manajemen memori, evaluasi ekspresi matematika, dan mekanisme undo pada editor teks. Queue adalah struktur data yang mengikuti prinsip first in first out atau FIFO dimana elemen yang pertama dimasukkan adalah yang pertama dikeluarkan. Queue digunakan dalam penjadwalan proses, antrian permintaan cetak, dan breadth-first search. Struktur data non-linear meliputi tree dan graph. Tree adalah hierarki node dengan satu node akar dan node-node lainnya bercabang menjadi anak-anak. Binary search tree adalah pohon biner dimana nilai node kiri lebih kecil dari node induk dan nilai node kanan lebih besar. Heap adalah pohon biner lengkap yang memenuhi properti heap dimana nilai node induk lebih besar atau lebih kecil dari anak-anaknya. Graph adalah kumpulan node atau vertex yang terhubung oleh edge. Graph dapat berarah atau tidak berarah dan dapat berbobot atau tidak berbobot. Algoritma graph penting meliputi depth-first search, breadth-first search, algoritma Dijkstra untuk jalur terpendek, dan algoritma Kruskal untuk minimum spanning tree. Hash table adalah struktur data yang memetakan kunci ke nilai menggunakan fungsi hash dan menawarkan kompleksitas rata-rata O1 untuk operasi insert, delete, dan search. Pemilihan fungsi hash yang baik penting untuk meminimalkan collision. Struktur data lanjutan meliputi disjoint set union untuk union-find, trie untuk pencarian string prefiks, dan segment tree untuk query range. Balanced tree seperti AVL tree dan red-black tree menjaga keseimbangan untuk memastikan kompleksitas logaritmik. Pemahaman struktur data dan kompleksitas algoritma dengan notasi Big O sangat penting dalam persiapan wawancara teknik dan pengembangan perangkat lunak skala besar.
