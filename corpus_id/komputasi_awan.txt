Komputasi awan atau cloud computing adalah model penyampaian sumber daya komputasi seperti server, penyimpanan, basis data, jaringan, perangkat lunak, dan layanan lainnya melalui internet. Model ini memungkinkan pengguna untuk mengakses dan menggunakan sumber daya komputasi sesuai kebutuhan tanpa harus memiliki dan mengelola infrastruktur fisik secara langsung. Komputasi awan menawarkan beberapa model layanan utama yang dibedakan berdasarkan tingkat abstraksi dan kontrol yang diberikan kepada pengguna. Infrastructure as a service atau IaaS menyediakan sumber daya komputasi virtual seperti mesin virtual, penyimpanan, dan jaringan yang dapat dikonfigurasi sesuai kebutuhan. Platform as a service atau PaaS menyediakan platform yang mencakup sistem operasi, runtime bahasa pemrograman, basis data, dan alat pengembangan sehingga pengembang dapat fokus pada pembuatan aplikasi tanpa mengelola infrastruktur. Software as a service atau SaaS menyediakan aplikasi lengkap yang dijalankan di infrastruktur penyedia dan diakses melalui browser web. Model deployment komputasi awan meliputi public cloud yang dimiliki oleh penyedia pihak ketiga, private cloud yang didedikasikan untuk satu organisasi, hybrid cloud yang menggabungkan public dan private cloud, serta multi-cloud yang menggunakan beberapa penyedia cloud sekaligus. Keuntungan utama komputasi awan meliputi skabilitas elastis yang memungkinkan penambahan atau pengurangan sumber daya secara otomatis sesuai permintaan, model pembayaran pay-as-you-go yang mengurangi biaya modal awal, akses global yang memungkinkan pengguna mengakses aplikasi dari mana saja, serta keandalan tinggi dengan redundansi di beberapa pusat data. Penyedia layanan cloud utama termasuk Amazon Web Services atau AWS, Microsoft Azure, dan Google Cloud Platform atau GCP. Layanan komputasi awan yang umum digunakan meliputi Amazon EC2 untuk komputasi, Amazon S3 untuk penyimpanan objek, AWS Lambda untuk komputasi tanpa server, Azure Functions, dan Google Cloud Storage. Komputasi tanpa server atau serverless computing adalah model eksekusi dimana penyedia cloud secara dinamis mengelola alokasi sumber daya dan pengguna hanya membayar untuk jumlah komputasi yang benar-benar digunakan. Kontainerisasi dengan Docker dan Kubernetes memungkinkan pengemasan dan orkestrasi aplikasi secara portabel di berbagai lingkungan cloud. Keamanan di komputasi awan mengikuti model tanggung jawab bersama dimana penyedia cloud bertanggung jawab atas keamanan infrastruktur cloud dan pengguna bertanggung jawab atas keamanan data dan aplikasi mereka. Migrasi ke cloud melibatkan strategi seperti rehosting yang memindahkan aplikasi tanpa perubahan, refactoring yang mengoptimalkan aplikasi untuk cloud, dan rebuilding yang merancang ulang aplikasi menggunakan layanan cloud native.
